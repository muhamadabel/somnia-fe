# 12 — UI Guideline

# Overview

This document defines the user interface principles for Dream Journal Analyzer.

The application should feel calming, modern, trustworthy, and easy to understand.

Users should focus on reflection rather than navigating a complex interface.

---

# Design Philosophy

The interface should communicate:

- Calm
- Clarity
- Trust
- Curiosity
- Personal Growth

The design should reduce stress rather than increase it.

---

# Visual Identity

The interface should appear:

- Modern
- Minimal
- Elegant
- Soft
- Friendly

Avoid visual clutter.

---

# Color Principles

Colors should support emotional comfort.

Primary Color

Used for primary actions.

Secondary Color

Used for supporting actions.

Success

Confirmation.

Warning

Attention.

Error

Failures.

Neutral

Backgrounds and text.

The exact palette should be selected during implementation.

---

# Typography

Typography should prioritize readability.

Recommended hierarchy:

Display

Heading

Subheading

Body

Caption

Code

Text should never feel crowded.

---

# Spacing

Use consistent spacing throughout the application.

Spacing should create visual breathing room.

Avoid overly compact layouts.

---

# Border Radius

Rounded components should communicate friendliness.

The radius system should remain consistent across:

Cards

Buttons

Inputs

Dialogs

Images

---

# Elevation

Elevation should be subtle.

Use shadows only when they improve hierarchy.

Avoid excessive depth effects.

---

# Buttons

Primary

Most important action.

Secondary

Alternative action.

Danger

Destructive actions.

Ghost

Low emphasis.

Disabled

Unavailable actions.

Loading

Pending operations.

Button styles should remain consistent across the application.

---

# Forms

Forms should:

- Validate clearly.
- Explain errors.
- Minimize typing.
- Support keyboard navigation.

Required fields should be clearly indicated.

---

# Cards

Cards should organize information into logical groups.

Examples:

Dream Card

Analysis Card

Report Card

Emotion Card

Community Card

Cards should not become visually overwhelming.

---

# Navigation

Navigation should always communicate:

Current Location

Available Actions

Back Navigation

Progress

Users should never feel lost.

---

# Icons

Icons should:

- Be simple.
- Be recognizable.
- Support labels.
- Remain stylistically consistent.

Icons should complement text rather than replace it.

---

# Empty States

Every empty state should:

Explain why it is empty.

Provide a next action.

Encourage user engagement.

Example:

"No dreams yet."

"Start recording your first dream."

---

# Loading States

Loading should use:

Skeleton Screens

Progress Indicators

Meaningful Feedback

Avoid blank pages.

---

# Error States

Error messages should:

Explain the problem.

Suggest recovery.

Provide retry actions.

Avoid technical jargon.

---

# Data Visualization

Charts should prioritize clarity.

Supported visualizations include:

Emotion Trends

Dream Frequency

Recurring Symbols

Weekly Progress

Charts should remain understandable for non-technical users.

---

# Responsive Design

Support:

Desktop

Laptop

Tablet

Mobile

Layouts should adapt naturally without removing important functionality.

---

# Accessibility

The interface should support:

Keyboard Navigation

Screen Readers

High Contrast

Readable Fonts

Focus Indicators

Accessible Forms

Accessibility should be considered throughout the design process.

---

# Motion

Animations should:

Support user understanding.

Remain subtle.

Avoid unnecessary distractions.

Motion should communicate state changes rather than decoration.

---

# UX Principles

Every screen should answer:

Where am I?

What can I do?

What happens next?

The interface should guide users naturally without requiring explanation.

---

# Future Design Evolution

The design system should remain flexible enough to support:

Dark Mode

Additional Themes

Personalization

Motion Preferences

New Components

Platform Expansion
